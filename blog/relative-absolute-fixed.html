CSS LAYOUT - RELATIVE VS. ABSOLUTE VS. FIXED POSITIONING

When you write HTML (a website's structure file) you put together many smaller elements. For example, the title of this blog post is an element. The positioning of these elements in CSS (a website's design guide) can be achieved using the position property.

Without any sort of positioning properties, elements would simply show in order they were written in the HTML document. The positioning property gives the ability to move elements, stick elements to the viewport (what you see of the website in your window) and change other element's behaviours when they encounter the subject elements.

The position property actually specifies the type of positioning method (group of characteristics and processes) used to position an element. Once the method is set, there will be further options available to adjust the element.

The methods that can be selected are:

  • position: static;
  • position: relative;
  • position: fixed;
  • position: absolute
  • position: sticky

The methods that we will focus on are relative, absolute and fixed.

position: relative;

An element with position: relative; is positioned relative to it's normal position within the normal flow of the HTML document.

With this property you will subsequently have access to the top, right, bottom, and left properties which will adjust the element away from it's normal position. Other content around it will not be adjusted or move to fill the gap left by the element if moved. Moving the element will not impact the positioning of other elements.

I'm relative!

The code:

Below is the code applied to the single div element above:

#example-relative {
position: relative;
left: 30px;
width: 100px;
height: 30px;
border: 3px goldenrod solid;
}

position: absolute;

An element with postion: absolute; is positioned relative to the whole HTML document and the parent element will behave as if the element is not there at all.

You will also have access to the top, right, bottom and left properties to adjust the position but an absolute element will not leave a gap in the document where it once was; it will be like it wasn't there. When other elements come into contact with a absolute element, other elements will not change their position.

I'm absolute's parent!
I'm absolute!

The code:

Below is the code applied to the parent div element with a single div element as above:

.example-parent {
width: 300px;
height: 100px;
border: 3px #0000ff6e dashed;
}

#example-absolute {
position: absolute;
left: 20px;
width: 100px;
height: 30px;
border: 3px goldenrod solid;
}

position: fixed;

An element with position: fixed; is positioned relative to the viewport. This means it always stays in the same place on the window even if the page is scrolled.

Adjusting the position of a fixed element is the same as adjusting an absolute element.

I'm fixed!

The code:

Below is the code applied to the single div that does not move when you scroll:

#example-fixed {
position: fixed;
bottom: 50px;
width: 100px;
height: 30px;
border: 3px goldenrod solid;
}

So....?

Positioning elements in this way can take a lot of time because each positioning rule has to be set to each element. It can also be rather labourous to make a website responsive.

Other systems of positioning that have been brought to CSS include Flexbox and CSS Grid. These systems transform making a responsive website to be almost automatic and it saves a lot of time by only requiring the rules to be set on the containing elements; at the worse-case, rules may have to be set to elements that fall as an exception to standard baked-in positioning rules.
